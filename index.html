Use DTOx to quickly create well formed DTOs

The Data Transfer Object is a lightweight, serializeable object for the purpose of transferring data. They can be valuable in the development of APIs where you would like to expose simple objects, and clean documentation. Writing them and their tests can be tedious at times, so use this package to quickly create them for your project.

Example

To create a DTO for our fictitious SludgeCo App, a command like this will do the trick:

$ dtox react dto 'SludgeCo\Acid\BurnyDTO' string:name int:burnLevel

This will create a BurnyDTO.php dto in the SludgeCo/Acid/ directory relative to where you run the command. Inside the file you will find a documented and PSR-2 compliant DTO class that implements the \Serializable interface.

To create the corresponding Unit Test for the DTO, use a command like this:

$ dtox react dto-unit 'SludgeCo\Acid\BurnyDTO' "Red Acid:name" 10:burnLevel

This will create a BurnyDTOTest.php test class in theSludgeCo/Acid/` directory relative to where you run the command. Note that instead of the variable type you put the test data that you want to include in the test class.

Installing

Add the package to your composer.json file:

{
    "require": {
        "jrobertfox/dtox": "dev-master"
    }
}

Then install:

$ composer install

Usage

The utility, right now, supports the react command to create a reaction with your "formula." The format is such:

$dtox react <FORMULA> <FULLY_QUALIFIED_CLASS_NAME> <VARIABLE_1> ... <VARIABLE_N>

Formula - the type of object you want to "react" ("dto" and "dto-unit" are currently supported)

Fully Qualified Class Name - The full name of your class, including the namespace.

Variables - Variables in the form <TYPE>:<NAME> that will be included in the DTO construction, for example string:name will create add a $name variable with the type string. For Test classes the form is <TEST_DATA>:<NAME>. This lets you build out your tests with realistic data.
